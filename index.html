WELCOME TO MMS WAF

A machine learning based web application firewall: The purpose of this project is to investigate the potential of using machine learning techniques for intrusion detection on an application level. The algorithms to be tested are classifiers trained on HTTP payload data which will take a payload as input and decide whether the input contained malicious code or not. WAF will protect from XSS,SQL and Shell-injections.

Steps (WorkFlow)

We had used CRISP-DM approach to build this machine learning model

1

Business Understanding

In this phase we had analyzed how the data flows in client server model. Followed by the understanding the various attacks on wild.
Varios injection vulnerabilities like XSS, SQL , Command injection and their types and analyzed the payload structure of each attack. 

2

Data Understanding

In this phase we had analyzed the requirements of features in data and stucture of data that we are going to prepare in next stage.

3

Data Preparation

After calculating the features of data set we move to actually prepare the data :
- We had collected the payoads of different attacks from various security data sources.
- We had performed feature engineering on our payloads to get most of that single string and had calculated features like length, min byte, max byte , mean byte, standard deviation of bytes,no of distinct bytes .. etc.  

4

Modeling

In this phase we started applying various machine learning algorithms on our data set. Since the WAF is a binary classification problem, we had applied various supervised classification techniques like logistic Regression, SVM, random forest, decision tree, KNN, XGBoost, Artificial neural network, lasso-lars,extra tress etc.

5

Evaluation

After Fitting any model the PERFORMANCE of model is calculated which involves : Confusion matrix, Accuracy, Precision, F1 Score,ROC - AUC Score ..etc. Based on the performance we had tried to maximise the accuracy of model via tunning the model. At last we will select one best model which is most accurate and its confusion matix should be best among all.

6

Deployment

This is the our personal favourite stage. In this phase we had exported the model via pickle library and generated a custom python script that we will be using for prediction. Later we had deployed this model on the aws cloud instance with website integration.
